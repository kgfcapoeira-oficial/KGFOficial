# Contribution Guidelines

Thank you for your interest in contributing to our project! These guidelines will help you set up your development environment and run the project locally.

## Setting Up Your Development Environment

1. **Clone the Repository**:
   
   ```bash
   git clone https://github.com/gcfilhosdofogo-hub/gc-filhos-do-fogo.git
   cd gc-filhos-do-fogo
   ```

2. **Install Dependencies**:
   
   Make sure you have [Node.js](https://nodejs.org/) installed. Then run:
   
   ```bash
   npm install
   ```

3. **Environment Variables**:
   
   Create a `.env` file in the root of the project and ensure it includes the necessary configuration variables. Refer to `.env.example` for guidance.

## Running the Project Locally

To start the development server, use:

```bash
npm start
```

Visit `http://localhost:3000` in your browser to see the project in action.

## Contributing to the Codebase

1. **Fork the Repository**:
   - Click on the 'Fork' button at the top right of the repository page.

2. **Create a New Branch**:
   
   ```bash
   git checkout -b your-feature-name
   ```

3. **Make Your Changes**:
   - Write clear and concise commit messages to describe your changes.

4. **Push Your Changes**:
   
   ```bash
   git push origin your-feature-name
   ```

5. **Open a Pull Request**:
   - Go to the 'Pull Requests' tab and click on 'New Pull Request'. Select your branch and submit your pull request.

## Code of Conduct

Please adhere to this project's [Code of Conduct](CODE_OF_CONDUCT.md) when contributing.

Thank you for contributing! Your efforts help improve the project for everyone!
